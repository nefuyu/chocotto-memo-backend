# ChocotoMemo (Backend API)

日常のちょっとしたメモを、フォルダごとに整理して管理できるREST APIサービスです。
バックエンドエンジニアとしてのポートフォリオ目的で作成しています。

## 1. 使用技術

* **言語/フレームワーク**: Java 25 / Spring Boot 4.06
* **データベース**: MySQL 8.4
* **ORM**: MyBatis
* **環境構築**: Docker
* **ビルドツール**: Maven
* **その他**: Lombok

## 2. 特徴・設計のこだわり

### 3層アーキテクチャの採用
保守性とテストの容易性を考慮し、Controller -> Service -> Repository -> Mapper の明確な責務分離を行っています。

### データの整合性と安全性の考慮
* **楽観的排他制御**: `update_count` カラムを導入し、複数ユーザーやタブからの同時編集によるデータの不整合（ロストアップデート）を防止しています。
* **論理削除の使い分け**: 
  * **メモ（Memo）**: 入力内容が多く消失リスクを最小限にするため「論理削除」を採用。
  * **フォルダ（Folder）**: 構成要素であり、削除後の再作成が容易なため「物理削除」を採用。

### RESTful APIへの意識
HTTPステータスコードの適切な使い分け（201 Created, 204 No Contentなど）や、Locationヘッダーの返却など、Web標準に準拠したインターフェースを設計しています。
